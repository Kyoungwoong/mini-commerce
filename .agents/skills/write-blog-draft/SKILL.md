# Skill: Write Blog Draft

Use this skill when converting implementation work into a blog draft or blog notes.

## Goal

Create a draft that explains:

```text
problem
→ implementation
→ result
→ limitation
→ next step
```

## Rules

- Write for Korean readers.
- Keep technical terms accurate.
- Explain why the step exists.
- Avoid pretending the learning implementation is production-ready.
- Include code snippets only when they help understanding.
- Do not make the post too long.

## Suggested Structure

```md
# Title

## 이번 글에서 할 것

## 왜 이 작업이 필요한가?

## 구현 방향

## 핵심 코드

## 실행 결과

## 지금 구조의 한계

## 다음 글 예고
```

## Output

If asked for blog notes, produce concise notes.

If asked for a full draft, produce a complete Korean draft.
